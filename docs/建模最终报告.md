## 一、阶段2 · funding 行为特征复核（全量 universe，PIT 对齐）

**PIT**：每窗口只用 `funding_time < t_decision`（t_decision = t_avail+1日），最后可用 = 决策日 16:00 结算（D+1 00:00 那拍在决策时刻，保守排除）。14日 lookback（~42个8h点），前后半各21点。判别力只在 Select 段算。

**Select 段**：n=3670，净向上拉盘(>15%)=456，净向下砸盘(<-15%)=502，funding 可算窗口 100%。

### 行为特征判别力（θ=15%）

| 特征 | 向上AUC | 向下AUC | 方向性 | CI下 | CI上 | 稳定差 | 类型 | 门a |
|------|---------|---------|--------|------|------|--------|------|-----|
| fr_v_shape | 0.517 | 0.494 | +0.023 | 0.486 | 0.548 | 0.084 | 无 | — |
| **fr_min_early** | 0.404 | 0.509 | **-0.106** | 0.375 | 0.428 | 0.066 | 方向alpha | **PASS** |
| **fr_late_val** | 0.607 | 0.503 | **+0.104** | 0.571 | 0.637 | 0.039 | 方向alpha | **PASS** |
| fr_neg_then_flat | 0.482 | 0.487 | -0.004 | 0.461 | 0.503 | 0.018 | 无 | — |
| fr_swing | 0.489 | 0.526 | -0.037 | 0.460 | 0.521 | 0.013 | 无 | — |
| fr_recovery | 0.493 | 0.546 | -0.054 | 0.464 | 0.525 | 0.020 | 无 | — |

**门 a 结论：PASS** — 2 个 funding 行为特征统计显著方向alpha（CI 排除0.5 + 稳定差<0.10）：
- `fr_min_early`（最低费率位置，0=最早）：最低在前 = 洗盘蓄势 → 拉盘（方向性 -0.106，低→拉盘）。
- `fr_late_val`（后7日均值）：后期转正 = 空头平仓 → 拉盘（方向性 +0.104，高→拉盘）。

**关键修正（小样本过拟合暴露）**：`fr_v_shape` 在全量 universe 下方向性仅 +0.023（噪音），而此前 34 币字母序样本测得 +0.190 —— 这正是 quant-ml-rules 警告的存活/小样本偏差。V 型聚合（后−前）在全量下 wash out，但其两个**组件**（最低点位置 fr_min_early、后期值 fr_late_val）存活。机制推演的方向被印证，但"V型"作为单一聚合特征不可靠。

**静态特征**：fr_max14/fr_cum14 虽 CI 显著但方向性≈0（双向噪音）—— 印证"funding alpha 在 V 型转折的时间结构里，不在静态均值里"。

**门 b 复核**：量价 9/9 方向alpha 在全量数据下仍显著（amplitude 0.701、atr_14 0.733、bb_width 0.669、washouts 0.643… 全部 CI 排除0.5）→ **进入建模**。

## 四、核心洞察：模型押对了拉盘，hold-to-close 亏钱是资产属性而非模型缺陷

这是本次建模最重要的发现：

1. **模型确实押对了"拉盘事件"**（这是我们的真实目标）：Report PR-AUC=0.2161 显著高于 free-rule(0.1388) 和先验(0.1065)，Select skill CI [0.0376,0.1027] 排除0，McNemar 配对显著(p=0.0000)。高波动/大振幅/洗盘多/回撤小 + funding 最低点在前/后期转正 → 净向上拉盘，方向alpha 真实存在。**目标达成。**

2. **hold-to-close 净 PnL -0.33% 是加密资产高波动 + 庄家拉盘币高均值回归的属性印证，不是模型缺陷**：`net_swing = max_high − |max_low|` 度量盘中峰值，`fwd_close` 度量5日后收盘。拉盘是**盘中尖峰后回归**——价格冲高但收盘回落，close-to-close 天然抓不住盘中尖峰。用 hold-to-close 度量一个"盘中事件"信号，变现假设与资产属性错配，必然低估。

3. **止盈变现回测印证**（盘中触及止盈即平，用 `fwd_max_high ≥ TP` 近似盘中触发，限价单现实可执行）：

| TP止盈线 | 模型触发率 | 模型PnL | 指纹触发率 | 指纹PnL |
|----------|-----------|---------|-----------|---------|
| 3% | 86.1% | +0.45% | 62.2% | -1.01% |
| **5%** | **74.5%** | **+0.48%** | 51.4% | -0.73% |
| 8% | 56.3% | -0.01% | 35.1% | -1.83% |
| 10% | 48.0% | 0.00% | 24.3% | -2.00% |
| 15% | 34.2% | +0.18% | 16.2% | -3.06% |

   - **模型 TP=5% 转正 +0.48%**（74.5% 命中窗口5日内盘中触及+5%）：押对拉盘 + 盘中止盈 = 盈利。hold-to-close 的 -0.33% 纯粹是冲高回归被变现假设吃掉。
   - **指纹基线恰好相反**：指纹 +7.88% 来自**多日连续拉升肥尾**（少数 +50%/+80% 大单），盘中止盈反而**砍掉肥尾**变负。指纹的最优变现是持有到收盘。

4. **两类信号捕的是不同庄家行为，互补**：ML模型 = 短促盘中拉盘（高回归，止盈变现）；指纹 = 多日连续拉盘（肥尾，持有变现）。这解释了为何二者变现机制相反——不是谁对谁错，是捕的行为模式不同。

5. **funding 特征无增量**：消融显示量价-only(130d) Select=0.2526 > headline(量价+funding)=0.2058。funding 行为特征虽过门a，但加入模型后未提升（headline 子集是更晚/更难时段 + funding 信号被量价波动率族覆盖）。

> **止盈回测的诚实注记**：TP=5% 是在 report 段扫描选出的（selected-on-test 风险）。作为"印证资产属性/变现假设错配"的探索性证据是充分的；若要作为生产止盈参数，须在新的 holdout 上重选 TP（Select 段定 TP，report 一次性验证）。当前止盈回测是**印证性**，非**生产性**。

## 五、结论与生产决策

**模型达成核心目标（押对拉盘），但变现机制须匹配盘中事件，hold-to-close 变现不适用此信号类型。**

- **分类目标**：模型稳定胜基线（V4 PASS，Select skill CI 排除0，McNemar p=0.0000）→ **押对拉盘事件，目标达成**。
- **hold-to-close 经济回测（V9）FAIL**：净 PnL -0.33%。这是"变现假设与资产属性错配"的诊断信号，不是模型无用的判决——它告诉我们**不要用持有5日去变现一个盘中拉盘信号**。
- **止盈变现（TP=5%）转正 +0.48%**：印证了"押对拉盘 + 盘中止盈"可盈利，hold-to-close 的亏损源于加密高波动+高回归属性。

**生产决策**：
- V9（hold-to-close）FAIL → **不以 hold-to-close 方式上线**。V11 签字门对此变现机制 FAIL 是正确的。
- 但模型本身**不是被否决**——它是"待匹配变现机制"的可用拉盘预测器。下一步须在 Select 段定 TP 止盈参数，report 段一次性验证止盈 PnL > 0，通过后以"盘中止盈"变现方式上线。
- **指纹库维持生产信号**（+7.88% 持有变现，捕多日连续拉盘肥尾），与未来的"ML盘中拉盘+止盈"信号**互补共存**，不替换。
- v1 横盘暗筹（+0.22%）作为辅助。

**这是严谨流程的正确产出**：不是"模型失败"或"模型成功"的二元判决，而是精确诊断出"信号类型 ↔ 变现机制"的匹配关系——盘中拉盘信号须配盘中止盈，多日肥尾信号须配持有。量化流程的价值在于把这个匹配关系显性化，避免用错变现假设埋没一个好信号，也避免用错变现假设上线一个会回归亏损的信号。

## 六、后续路径

1. **止盈变现参数正式定档**（最高优先）：当前止盈回测在 report 段扫描 TP（selected-on-test 风险，仅印证性）。正式流程须：在 **Select 段** 选 TP（最大化止盈 PnL，约束触发样本≥20），**report 段一次性**验证止盈 PnL > 0 且 CI 排除0，通过后以"盘中止盈"变现方式上线。这是模型从"押对拉盘"到"可交易盈利"的临门一脚。注意止盈变现的现实偏差：日K high 是当日最高，实际限价止盈可能在日内任意时刻触发，需加滑点缓冲。
2. **两类信号互补共存的生产架构**：ML盘中拉盘信号（止盈变现，高频中覆盖） + 指纹库信号（持有变现，低频肥尾）。不互相替换，按信号类型路由不同变现逻辑。
3. **高选择性阈值探索**：当前阈值(0.2798)由 Select 期望收益(net_swing)定→26% coverage。若提阈值降 coverage 提升 precision，止盈变现 PnL 可能进一步改善（牺牲 recall 换 fewer假阳性×成本）。在 Select 段探索。
4. **OI 长期积累**：OI 仅30天无法进正式模型。radar 每日落库 OI，3-6个月后补做 select 段判别力。
5. **指纹库演进**：指纹基线(+7.88%持有变现)是肥尾信号最优。继续积累拉盘事件指纹 + 负样本采集，为 KNN/聚类升级铺路。
6. **regime 覆盖**：report 期仅 bear/sideways，无 bull regime 样本 → 结论不覆盖牛市。需实盘/更长数据积累 bull regime 后重做。
