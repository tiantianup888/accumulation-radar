# 庄家拉盘特征挖掘方案（数据科学严谨版）

> 起因：当前 6 个签名是 4 个币归纳+阈值拍脑袋的产物，属"先验臆想特征"。
> 本方案改为数据驱动的特征挖掘管线，严格对齐 quant-ml-rules skill 的非协商项。
> 先确认方案，再实现。

---

## 一、当前方法诊断（要被取代的）

| 病灶 | 具体 |
|------|------|
| 特征先验 | 6 签名看 4 币归纳，4 币验证=自答 |
| 阈值拍脑袋 | 量>2×/上影>5%/跌<-8%/收敛<0.7/止跌-1.5~1%/pos<0.3 全无出处，且布尔化丢连续信息 |
| 组合主观 | 0.4/0.3/0.3 权重 + "收敛∧止跌∧低位"门控手定 |
| 无特征筛选 | 未系统评估判别力 |
| 无基线 | 未在统一协议下对比随机/v1 |
| selected-on-test | 调阈值时看了样本外 |

## 二、标签构造（最先定，一切围绕它）

严格时序因果（skill §1/§2/§3）：

```
窗口 = [i-13, i] 14 个交易日（日K收盘）
t_feature   = 第 i 日（窗口末根 K 线）
t_available = 第 i 日收盘后（当夜才能知道这根 K 线）
t_decision  = t_available（收盘后决策，不偷看未来）
t_target    = 未来 5 日 [i+1, i+5] 的最大涨幅 max_high
t_realized  = 第 i+5 日收盘

硬约束：t_available ≤ t_decision < t_realized
```

**标签 Y**（二分类，阈值 θ 待定）：
- `Y[i] = 1{ max_high[i+1..i+5] / close[i] - 1 > θ }`
- θ 候选：+15% / +20% / +30%（用 Select 角色选，不在 Report 上选）
- 同时保留回归版 `r[i] = forward_5d_return` 供回归模型用

**重叠标签处理（skill §4，非协商）**：5 日前瞻标签若窗口步长≤5 日则重叠，必须 purge+embargo：
- Purge：删掉训练样本中标签窗口与测试标签窗口相交的
- Embargo：每个测试块后丢弃约 5 日的训练候选
- 或：窗口步长直接取 5 日（=前瞻窗口），消除重叠，代价是样本减半——优先用这个更干净

## 三、特征池（广撒网，不先验筛，连续值不固化阈值）

对每个 14 日窗口提取一大组候选特征。**关键：之前 6 签名的连续版也放进去（不布尔化），但只当候选，留不留由数据决定。**

| 类别 | 候选特征（连续值为主） |
|------|----------------------|
| 收益统计 | 1/3/5/7/14 日收益率、收益均值/方差/偏度/峰度 |
| 波动率 | 多窗口 ATR(3/5/7/14)、振幅、波动率变化率、波动率收敛比(近3/前7)连续值 |
| 量能 | 量比均值/峰值/方差/趋势斜率、量价相关、放量日占比、OBV 斜率 |
| 位置 | 相对 14 日高低位 pos 连续值、均线偏离、布林带位置 |
| 形态 | 转折点数、上行/下行天数、最大回撤、峰谷比、加速/减速 |
| 时序 | 一阶自相关、Hurst 指数 |
| 技术指标 | RSI、MACD hist、ATR%、BB 宽度（标准化） |
| 签名连续版 | test_pulse 计数、washouts 计数、converge_ratio、pos、（不固化阈值） |
| 合约端(有则用) | 资金费、OI 5 日斜率（注意 OI 历史窗口受限） |

**不做**：不先验剔除、不手定组合、不固化阈值。最终留哪些由 §四筛选决定。

---

## 四、特征筛选（数据说话，回答"哪个真有判别力"）

1. **单变量判别力**：每个特征对 Y 算 AUC / 互信息 / IV，排序。客观回答"止跌/收敛/探底到底有没有判别力"。
2. **共线性剔除**：特征间相关 >0.9 的留判别力强的那个。
3. **稳定性**：分时段算判别力，跨时段稳定的才留（防某段偶然）。
4. 这一步**只看 Select 角色数据**，不碰 Report。

---

## 五、模型阶梯（skill §15，不跳级）

```
free rule / 随机选币 / v1 横盘暗筹  →  logistic 回归  →  LightGBM  →  (可选) DTW-KNN
```

| 层 | 模型 | 作用 |
|----|------|------|
| 基线 | 随机选币、v1 横盘暗筹、当前指纹模板匹配 | 任何模型打不过基线=无结果（skill §5） |
| L1 | Logistic 回归（L1/L2） | 可解释基线，看系数方向+显著性，验证"特征方向是否符合假设" |
| L2 | LightGBM | 特征重要性 + SHAP，发现非线性交互（"收敛 AND 止跌"这类条件让树去发现，不是我手定） |
| L3 可选 | DTW-KNN（shape 作距离） | 低维形态类比，需要先 PCA 降维 |

**首选 L1→L2**。样本量不支持深度学习（skill §11 小样本诚实）。

## 六、评估协议（skill §6/§7，反 selected-on-test）

```
时间轴: |-- train --|-- select(val) --|-- report(test, frozen) --|
         调参/特征筛选只在 select        只看一次，never 反哺
```

- **Walk-forward 或单 frozen holdout**（90 天数据短，倾向单 holdout：前 60 天 train+select，后 30 天 report）
- 特征筛选、超参、阈值、early stopping 全在 Select 角色定
- Report 只评估一次；若回头改特征=test 烧掉，需新 holdout
- 三个角色数据严格不相交，purge+embargo 处理重叠

---

## 七、度量（skill §8/§9，匹配交易目标）

不用 accuracy 当主指标（标签极度不平衡）：
- **PR-AUC**（不平衡下的判别力）
- **召回 @ 固定精度**（如 precision=0.3 时的 recall，对应"预警里多少真拉盘"）
- **期望收益 / 肥尾捕获率 / RR / 跌>10% 占比**（与项目一致）
- **Skill score = 1 − loss_model / loss_baseline**（必须打过基线）
- 小样本报 block bootstrap 置信区间（skill §11）
- N、日期范围、regime 随每个数字披露

## 八、关键 guardrails 自检表（skill 对齐）

| 规矩 | 本方案如何满足 |
|------|---------------|
| 时间因果不 shuffle | 标签严格 forward，按 t_available 排序，时序 CV |
| t_available 显式 | §二已定义四元组 |
| 标签/特征不泄漏 | 标签用未来，scaler/筛选只 fit 在 train fold |
| 重叠 purge+embargo | §二窗口步长=前瞻窗口或 purge+embargo |
| 基线强制 | §五含随机/v1/当前指纹三个基线 |
| Select/Report 分离 | §六三角色不相交 |
| 阈值=selection | 在 Select/calib 定，不在 Report |
| 度量匹配目标 | §七用 PR-AUC+期望收益+肥尾，非 accuracy |
| 小样本诚实 | 报置信区间，≥0.5pp 当噪音 |
| 模型阶梯不跳级 | logistic→LightGBM，不跳深度学习 |
| regime 诚实 | 按 bull/bear/chop 分段报，不混报 |

## 九、执行计划（确认后）

1. **建特征矩阵管线** `feature_mining.py`：
   - `build_labels()`：forward 5 日 max_high 标签
   - `build_features()`：§三特征池（连续值）
   - `availability_mask()`：t_available 对齐
   - `split_temporal()`：train/select/report + purge+embargo
2. **特征筛选报告**：单变量 AUC/MI/IV 排序 + 共线性 + 稳定性，输出 `docs/特征判别力报告.md`。**先看这步结果再建模**——如果连单变量都没特征显著，就别建模了（skill：打不过基线就停）。
3. **基线先行**：随机/v1/当前指纹在 report 段的期望+肥尾。
4. **logistic → LightGBM**：SHAP 解释，看数据发现的组合是否合理。
5. **定稿**：打过基线且 Select 稳健的配置才上线，否则停。
