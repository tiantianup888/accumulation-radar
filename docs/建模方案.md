## 阶段0 · 可用性、PIT对齐与样本边界

### 0.1 可用性时间表（每个数据源各自的 t_available）
| 数据源 | 粒度 | t_feature | t_available（可知晓时刻） |
|---|---|---|---|
| 日K klines | 日 | 日 D 的 [D 00:00, D+1 00:00) | D+1 00:00（收盘后） |
| funding | 8h | 结算时刻 F | F（结算即公布） |
| OI（openInterestHist） | 日 | 日 D | D+1（次日可得） |

**行索引 = t_decision**。一个 14日 lookback 窗口在"最后一根可用日K = 日 D"时：
- `t_decision = D+1 00:00`（窗口描述截至 D 收盘，D+1 才能决策）
- 特征只用 `t_available ≤ t_decision` 的数据 → 日K ≤ D；funding ≤ D+1 00:00；OI ≤ D（OI 次日才得，故只能用到 D-1）
- `t_target` = [t_decision, t_decision+5日) 的 net_swing
- `t_realized` = t_decision + 5日
- **硬规则**：`t_available ≤ t_decision < t_realized`（§2）

### 0.2 PIT join（§13）
- 量价/funding/OI 各按自己的 t_available join 到窗口，**不按日历等值**。
- funding 行为特征只用 `funding_time ≤ t_decision` 的 funding 点；OI 特征只用 `ts ≤ t_decision - 1日`。

### 0.3 标签
- `net_swing = fwd_max_high − |fwd_max_low|`（净幅度，5日前瞻）。
  - 正=净向上拉盘，负=净向下砸盘，≈0=双向波动噪音（排除波动率聚集假象）。
- 阈值 θ=0.15（最严格，decision door 用），备选 0.10/0.05 做敏感性。`y_θ = 1{net_swing > θ}`。

### 0.4 窗口与防泄漏
- 14日 lookback，步长3日，标签窗5日。
- **purge**：删除 train 中标签窗 [t_decision, t_decision+5日] 与 select/report 标签窗重叠的样本。
- **embargo = 5日**（=1标签窗，protocols §2；原2日太短已修正）：train→select、select→report 边界各留5日缓冲。
- 相邻窗口标签重叠 (5−3)=2日 → purge 必须处理，不能当独立样本。

### 0.5 时间分割（单holdout + nested CV stress）
- **主协议**：单 frozen holdout，时间序 Train(60%)/Select(20%)/Report(20%)，**严禁 shuffle**。
- 单holdout可用条件（§6）：(a) 选择绝不看 report；(b) **试验预算封顶**（logistic ≤16配置，LightGBM ≤24配置，声明 trial count）；(c) 报 block-bootstrap CI。
- **稳健性stress**（protocols §10）：冻结后跑 **3-block 时间序列 nested CV**（外层3个时间块，内层Select搜索），作为次要报告，确认单holdout数非偶然。若 nested 与单holdout结论冲突，以 nested 为准并降级结论。
- 若搜索范围扩大 → 必须切回 nested CV 为唯一主协议。

### 0.6 样本边界与存活偏差（§10/§13，诚实）
- **研究span = funding 覆盖段**（~90天）：funding 仅90天，覆盖不到最早40天 → **headline 模型只用 funding 完整覆盖 lookback+标签 的窗口**（不补全/不前填/不全局填充）。量价-only 全130天作为**消融**（验证 funding 增量价值），不作 headline 对比（§10：跨模型对比须同 span）。
- **universe-as-of**（§13）：窗口剔除该 symbol 在 t_decision 时不存在/历史不足14日的样本（新币上市不足lookback）；**退市币**只保留其在真实交易期内的窗口，且要求标签窗 [t_decision, t_decision+5日] 完全落在真实交易期内（退市后冻结K线不可作标签）。
- **存活偏差修正（已实施）**：
  - 数据源：当前 exchangeInfo 取所有 USDT PERPETUAL（含 status=SETTLING），+ Wayback Machine 2个 exchangeInfo 历史快照（2026-02-17/05-10）交叉验证退市时间。
  - 补拉 **122 个 SETTLING 退市币** 的 klines/funding/OI → DuckDB。
  - **冻结K线清洗**：SETTLING 币进入结算后发布 high=low 且 vol=0 的冻结价K线（非真实交易），已全量删除（14184行）。`data_audit.py` 增冻结K线检查。
  - **`symbol_meta` 表**（universe-as-of 核心）：记录每币 status/onboard_date/last_kline_date(真实末日)/source。`DuckDBStore.universe_as_of(t_decision)` 提供按决策时刻的合法universe。
  - 结果：全量654币（TRADING 530 + SETTLING 37 + DELISTED_PREWINDOW 85 + PENDING 2）；**有真实日K 567币**；**可建模universe 559币 = 530 TRADING + 29 窗口期退市SETTLING**（≥19根真实日K，可形成 lookback14+label5 窗口）。29个退市币中19有funding、10无funding（早于funding窗口结算，funding特征NaN由缺失策略处理）。
  - **残留局限（诚实）**：Wayback 仅2个快照时点，完全退市（从 exchangeInfo 移除）且未被快照捕获的币仍可能遗漏；当前修正覆盖了"窗口期在交易、现已SETTLING"的币，这是最主要的存活偏差源。退市币的真实交易末日 = funding末日 ≈ 进入SETTLING 时刻。

## 阶段3 · 基线先行（强制，禁止跳过，§5/protocols §7）
在 **report 段、同一 report 索引**上报四个基线：
- **基线A · 先验**：始终预测多数类（PR-AUC=正样本率，acc=多数类率）。
- **基线B · free-rule**：最强单特征阈值门（如 amplitude_14d > select 段中位数）—— skill §5 要求的"简单公开规则"。
- **基线C · v1 横盘暗筹**（当前生产信号 `sideways_acc`）。
- **基线D · 指纹库模板匹配**（当前生产信号 `fingerprint`，锁定配置 sim≥0.65+收敛+止跌+低位）。
- 报 skill score = 1 − loss_model/loss_baseline（metrics.md）。
- **门**：模型必须 skill score > 0（CI 不跨0）才考虑替换生产信号；跑不过基线则保留基线/停产。

## 阶段4 · 模型阶梯（不跳级，§15/models.md §0A）

### 4.1 模型1 · Logistic 回归（线性天花板，可解释基线）
- **§0A scenario 行**：weak-signal tabular alpha / interpretable gate / 容量受限生产系统。
- 用10–16特征（经共线性处理），L2 正则。
- **scaler fit 在 train-fold only**（§4.2，禁全样本标准化）；`C` 在 Select 网格选。
- `scale_pos_weight` 仅来自 train 段 base rate。
- 报 train/Select 系数为诊断（冻结后），不在 report 上反复看系数调特征。
- 输出 `n_params` 作复杂度代理。

### 4.2 模型2 · LightGBM（发现非线性组合）
- **§0A scenario 行**：nonlinear interactions among tens of tabular features, N effective large。
- **仅在 logistic Select skill > 0 且残差呈非线性**时升级（partial dependence / 残差结构判断）。
- 训练协议（models.md §1.2-1.8）：
  - `objective=binary`，`eval_metric` 对齐 Select 目标（PR-AUC/logloss）。
  - **early stopping 监控 Select only**，`best_iteration` 用于 predict；`num_boost_round` 上限大。
  - 复杂度用 `num_leaves` + `min_data_in_leaf` 控制（小N/重叠→调高 min_data）。
  - `scale_pos_weight` 仅 train base rate；`subsample`/`colsample` 0.6–0.9。
  - **monotone constraints**：方向已知特征加约束（amplitude/washouts→正，max_drawdown→负）。
  - 阈值在 Select/calib 调，report 只评一次。
- **特征重要性**：用 **permutation importance**（purged-fold）作 claim，gain 仅参考；**SHAP 仅诊断，不在 report 上挖了再回改 report**（§1.6）。
- 需装 `sklearn` + `lightgbm`。

### 4.3 跨模型对比公平性（models.md §10）
- logistic vs LightGBM：**同特征、同标签、同 Select/Report span、同基线、同搜索预算类**。
- 胜者 = Select skill 最佳；report 仅对冻结胜者评一次。
- Select skill 在噪声内 → 选更简单模型（logistic）。

## 阶段5 · 阈值=selection（不在report调，§7/protocols §6）
- 在 **Select/calib 段**选阈值：最大化期望收益，约束最大回撤。
- 参考 Fβ（低胜率肥尾策略 → β>1 偏 recall，容忍多 FP 捕肥尾），但**运营阈值用期望收益+回撤约束**定。
- report 段用 Select 定的阈值，**只评一次**。
- 若 calib 单类 → 回退默认阈值0.5并记录。

## 阶段6 · 度量（非accuracy，§8/metrics.md）
- **PR-AUC**（正样本少，不用 ROC-AUC/accuracy）。
- **期望收益** = 命中窗口 net_swing 均值 × 命中率。
- **肥尾捕获率** = top5% 涨幅占命中池比例。
- **最大回撤 + RR**（期望/最大回撤）。
- **coverage** = gate 允许交易的时段占比（§8）。
- **base rate** `P(y=1)` 与条件 `P(y=1 | key_state)`（如高波动组）。
- **skill score** = 1 − loss_model/loss_baseline（对每个基线）。
- **每个headline数附 N + 日期范围**（metrics.md 命名规则）。
- **regime 分段报**：按 BTC 30日趋势分 bull/bear/sideways 分别报（§10）。
- **数据集 regime 声明**：130天窗口处于何种 regime（整体 bull/bear/chop），不overclaim泛化性。
- **train→report 分布漂移检查**：label base rate、特征 mean/std 在 train vs report 的差异；漂移大 → 缩短训练窗/衰减加权（在 Select 选）。

## 阶段7 · 小样本诚实（§11/metrics.md）
- report 段 ~2857 窗口，正样本率 ~10% → **block bootstrap**（block 长度 ≥ 标签窗5日，因步长3日重叠）。
- **配对显著性**：模型 vs 基线用 **McNemar** 或 skill score 的 block bootstrap（不报裸绝对差）。
- **试验计数**：记录 trial count，启发式惩罚 √trials × se；多试验下 ≥0.5–1pp delta 视为噪声除非过显著性。
- CI 过宽无法下结论 → 明说"样本不足，实盘继续积累"，不强行出结论。

## 阶段8 · 经济回测与上线（§9/§14，生产系统必做）
> 项目是生产系统（radar 推信号），模型须过经济回测才替换生产。

### 8.1 经济回测（冻结后，report 段）
- prediction → **gate**（交易/不交易），不是连续仓位。
- 成本：手续费 + funding（持仓跨8h结算）+ 滑点模型。
- 回测用**与实盘相同的可用性延迟**（日K次日才得，t_decision=D+1）。
- 多 regime 切片 stress + no-trade zone + 容量假设。
- 不因分类准确率升而 PnL 跌就上线（§9）。

### 8.2 Live parity（§14）
- 同特征定义、同 lag、同缺失策略；不偷未完成K线。
- model card：决策频率、取消规则、陈旧数据规则、kill-switch→回退基线。
- 缺失特征显式处理（默认/跳过交易/fail loud，§13：禁静默特征集降级）。

### 8.3 上线路径（protocols §10）
research freeze → paper trade/shadow（对比特征分布+延迟）→ 小仓位实盘 → 持续残差监控 vs 基线。
仅当 nested/forward 评估下 report skill 存活 **且** 过成本才promote。

## 阶段9 · 建模后验证流程（保证结果正确可信）
> 目的：建模完成后，**必须**过本流程全部门才允许 claim 任何结论 / 替换生产信号。任一门失败 → 修复或停止，不得跳门上线。
> 顺序按 quant-ml-rules「How to Review」硬优先级：泄漏 → 污染 → 重叠 → 基线 → 模型族 → 度量 → 漂移 → live parity → 复现。
> 落地：可自动化部分写进 `verify_model.py`（阶段9脚本），每门输出 `Vx=PASS/FAIL`；人工判断项标注「人工」。

### V1 · 泄漏复审（look-ahead / availability / label alignment，硬失败优先）
- **检查**：① 重跑 `data_audit.py` 确认数据无新 gap/异常；② 特征-标签泄漏自检：每个特征列的 t_available ≤ t_decision，标签 net_swing 仅用 [t_decision, t_decision+5日) 且不含 t_decision 当根K线；③ scaler/PCA/imputer 确认 fit 在 train-fold only（禁全样本）。
- **通过条件**：零泄漏；`t_available ≤ t_decision < t_realized` 全样本成立。
- **验证方法**：`verify_model.py` 扫描特征矩阵的 t_decision vs 标签窗起止，断言无前瞻；人工核对 funding/OI 的 PIT join 代码。
- **失败处理**：硬失败，立即停，修代码后重跑阶段0–8。

### V2 · 三分离合规审计（selected-on-test，§6）
- **检查**：① 实验日志中 keep/discard 决策只引用 `hpval_*`（Select 指标），不引用 `report_*`；② early-stop 监控集 ⊆ Select；③ 阈值/calibrator 在 Select/calib 调，report 只评一次；④ `selected_on_test` 标志=0（若曾看 report 调参则=1 且不得 claim OOS）。
- **通过条件**：所有选择决策的输入指标均为 Select 角色；`selected_on_test=0`。
- **验证方法**：`verify_model.py` 扫实验日志 METRIC 行，断言 keep/discard 依据列不含 `report_`；人工复核特征增删动机。
- **失败处理**：若 report 被看过且影响了选择 → report 已污染，降级为 `selected_on_test=1`，须用新 holdout 或 nested CV 重做。

### V3 · 重叠 / purge / embargo / 有效样本量（§4/§11）
- **检查**：① purge+embargo=5日 确实执行（train→select、select→report 边界各缓冲5日）；② 相邻窗口标签重叠(5−3=2日) 已被 purge 处理；③ report 有效独立块数 N_eff（块长度≥5日）≥ ~100，否则报「样本不足」。
- **通过条件**：purge/embargo 落地；N_eff 报出且≥阈值，或显式声明上偏。
- **验证方法**：`verify_model.py` 重放分割，断言边界缓冲存在；block-bootstrap 估 N_eff。
- **失败处理**：N_eff 过小 → 不下结论，继续实盘积累（§阶段7 小样本诚实）。

### V4 · 基线诚实（§5，强制）
- **检查**：四基线（先验A / free-rule B / v1 C / 指纹D）在**同一 report 索引**上全部报出；模型 skill score = 1−loss_model/loss_baseline 对每个基线的 **block-bootstrap CI 排除 0**。
- **通过条件**：模型 skill score > 0 且 CI 不跨 0（对最强基线亦成立）。
- **验证方法**：`verify_model.py` 比对模型 vs 每基线的配对 block-bootstrap / McNemar。
- **失败处理**：跑不过最强基线 → 保留基线/停产，不替换生产信号（§5：beats no baseline = no result）。

### V5 · 模型族协议（§15/models.md）
- **检查**：① logistic scaler fit train-fold only、`C` 在 Select 选；② LightGBM early-stop 监控 Select、`best_iteration` 用于 predict、`scale_pos_weight` 来自 train base rate、monotone 约束加在方向已知特征；③ permutation importance 在 purged-fold 上算作 claim，gain 仅参考，SHAP 仅诊断不回改 report；④ 跨模型对比同特征/同标签/同 span/同基线/同预算。
- **通过条件**：所选模型族每条 hard rule 满足；未跳级（先 logistic 线性天花板再 LightGBM）。
- **验证方法**：`verify_model.py` 读模型 config 断言 scaler/early-stop/monotone 标志；人工核对升级动机（残差非线性证据）。
- **失败处理**：违反任一 hard rule → 修协议重训；跳级无理由 → 退回 logistic。

### V6 · 度量匹配（§8）
- **检查**：① 主指标为 PR-AUC（非 accuracy/ROC-AUC，正样本少）；② 度量命名带角色前缀（`hpval_`/`report_`）；③ 交易目标必报期望收益+最大回撤+RR+coverage+肥尾捕获率；④ 每个 headline 数附 N + 日期范围。
- **通过条件**：度量集与命名规范齐全；无裸 accuracy claim。
- **验证方法**：`verify_model.py` 断言输出键集合 ⊇ 规范要求集。
- **失败处理**：缺度量补报；命名不符改名。

### V7 · 漂移 / regime / 小N / 多重试验（§10/§11）
- **检查**：① 报 by-regime（BTC 30日趋势 bull/bear/sideways）分段指标；② 声明数据集整体 regime，不 overclaim 泛化；③ train→report label base rate / 特征 mean-std 漂移量化；④ trial count 记录，√trials×se 惩罚后 delta 仍显著；⑤ block-bootstrap CI 报出。
- **通过条件**：by-regime 报出；漂移大时已缩窗/衰减（Select 选）；CI 不跨结论线。
- **验证方法**：`verify_model.py` 算 train vs report base rate/特征分布 KS/PSI；人工判 regime 划分合理性。
- **失败处理**：漂移致命 → 缩训练窗重做；小N CI 过宽 → 不下结论。

### V8 · 复现性（§13）
- **检查**：① 确定性 seed 记录；② 库版本（numpy/sklearn/lightgbm）记录；③ 冻结 config 重跑 → report 数值逐位一致（容差 < 1e-6）；④ 失败 loud（如 OI/BTC 副表为空时 config 依赖它 → 报错非静默降级）。
- **通过条件**：冻结 config 可复现；版本/seed 入档。
- **验证方法**：`verify_model.py` 用冻结 config 二次运行，diff report 指标。
- **失败处理**：不可复现 → 固定 seed/版本后重冻；静默降级 → fail loud 修代码。

### V9 · 经济回测（§9，生产系统必过）
- **检查**：① prediction→gate（交易/不交易）映射明确；② 成本含手续费+funding（跨8h结算）+滑点；③ 回测用与实盘相同可用性延迟（日K次日才得）；④ 多 regime 切片 stress + no-trade zone + 容量假设；⑤ 不因 accuracy 升而 PnL 跌就上线。
- **通过条件**：report 段净 PnL/Sharpe/最大回撤 优于 flat（不交易）且过成本；regime 切片无致命崩溃。
- **验证方法**：`verify_model.py` 跑经济回测模块，断言 net PnL > 0（含成本）+ 最大回撤可接受。
- **失败处理**：经济回测亏 → 不上线（即便分类指标好看）。

### V10 · Live parity（§14）
- **检查**：① 研究特征定义/lag/缺失策略与生产 radar 一致；② 不偷未完成K线；③ model card 含决策频率/取消规则/陈旧数据规则/kill-switch→回退基线；④ 缺失特征显式处理（默认/跳过/fail loud，禁静默降级）。
- **通过条件**：model card 齐全；kill-switch 定义；研究=生产路径同构。
- **验证方法**：人工核对 radar 集成代码特征定义 vs 研究特征定义逐项一致。
- **失败处理**：live parity 不符 → 修生产集成或修研究特征定义使其一致。

### V11 · 签字门（sign-off，全部门通过才 promote）
- **规则**：V1–V10 全 PASS → 冻结 config → paper/shadow trade → 小仓位实盘 → 持续残差监控 vs 基线。
- 任一 FAIL → 修复后从失败门重跑后续；不得跳门。
- **输出**：`verify_model.py` 生成 `docs/建模验证报告.md`，列每门 PASS/FAIL + 证据 + N/日期/seed/版本。

## 术语注解与规范（量化/ML/统计学术语标准化）
> 本节为全文术语的权威定义与**本项目规范用法**。建模/报告/代码须遵循此规范，禁止歧义使用。
> 依据列 = quant-ml-rules 非协商项编号。

### A. 时间因果与样本边界
| 术语 | 全称/中文 | 定义 | 统计学含义 | 本项目规范用法 | 依据 |
|---|---|---|---|---|---|
| **PIT** | Point-in-Time | 数据"在某个真实时刻可知晓"的版本 | 防止用未来才公布/修订的值当历史特征 | 每源按自身 t_available join，不按日历等值 | §2/§13 |
| **t_available** | 可用性时刻 | 某条数据最早能在生产中知晓的墙钟时刻 | 决定特征能否合法进入决策 | 日K=D+1 00:00；funding=结算时刻；OI=D+1 | §2 |
| **t_decision** | 决策时刻 | 模型允许发信号的时刻 | ≥ 所有特征 t_available 的 max | =窗口最后一根可用日K次日 00:00 | §2 |
| **t_realized** | 标签实现时刻 | 目标结果可知晓的时刻 | 必须 > t_decision | = t_decision + 5日 | §2 |
| **purge** | 清洗 | 删除标签窗与验证/测试标签窗重叠的训练样本 | 消除重叠标签导致的边界泄漏 | 删 train 中标签窗与 select/report 重叠者 | §4 |
| **embargo** | 禁运 | 测试块后丢弃一小段后续训练候选 | 防标签窗尾缘信息渗入下一段 | =5日（=1标签窗），边界各留5日 | §4 |
| **walk-forward** | 滚动前进 | 逐点用历史训练、在下一时刻预测 | 模拟真实时序决策 | nested stress 用；主协议单holdout | protocols §B |
| **nested CV** | 嵌套交叉验证 | 外层报分、内层搜索，外层不看内层赢家 | 防选超参污染泛化估计 | 3-block 时间序列 nested 作 stress | §6/protocols §C |
| **block bootstrap** | 块自举法 | 以时间块为单位重采样估CI | 处理序列自相关，非独立样本 | 块长≥5日（标签窗），报 report CI | §11 |
| **单holdout** | 单冻结测试集 | 时间序切 Train/Select/Report，report 只看一次 | 小搜索预算下的简化协议 | 主协议，须试验预算封顶+报CI | §6 |

### B. 标签与度量
| 术语 | 全称/中文 | 定义 | 统计学含义 | 本项目规范用法 | 依据 |
|---|---|---|---|---|---|
| **net_swing** | 净幅度 | fwd_max_high − \|fwd_max_low\| | 净向上拉盘幅度，排除双向波动噪音 | 标签连续值，阈值化 y_θ=1{>θ}，θ=0.15 | §3/用户决策 |
| **AUC** | Area Under ROC Curve | ROC 曲线下面积 | 二分类判别力，0.5=随机 | 单变量判别力用，**不作多基线主指标** | §8 |
| **PR-AUC** | Precision-Recall AUC | PR 曲线下面积 | 正样本少时比 ROC-AUC 诚实 | **多基线对比主指标**（正样本率~10%） | §8/metrics |
| **ROC-AUC** | — | ROC AUC | 不受类不平衡影响，易高估 | **禁用为 headline**（正样本少会虚高） | §8 |
| **base rate** | 基率 | P(y=1) | 正样本占比 | 报每段 base rate + 条件 P(y=1\|state) | §8 |
| **coverage** | 覆盖率 | gate 允许交易的时段占比 | 信号可用性 | 报 report coverage | §8 |
| **skill score** | 技巧分 | 1 − loss_model/loss_baseline | 相对基线的增量 | 对每基线报，CI 排除0才 claim | §8/metrics |
| **Fβ** | F-beta score | (1+β²)·P·R/(β²P+R) | β>1 偏 recall | 低胜率肥尾→β>1，但运营阈值用期望收益定 | §8 |
| **McNemar** | 麦克尼马尔检验 | 配对二分类显著性 | 模型 vs 基线配对比较 | 配对显著性用，不报裸绝对差 | §11 |
| **肥尾捕获率** | fattail capture | top5% 涨幅占命中池比例 | 捕获极端正收益能力 | 低胜率肥尾策略核心度量 | §8/用户决策 |
| **RR** | reward/risk | 期望收益/最大回撤 | 风险调整回报 | report 报 | §8 |
| **最大回撤** | max drawdown | 峰到谷最大跌幅 | 下行风险 | 经济回测+report 报 | §8/§9 |
| **期望收益** | expected return | 命中窗口 net_swing 均值×命中率 | 期望值视角 | **度量优先级第1** | §8/用户决策 |

## 风险控制清单（quant-ml-rules 对齐）
- **look-ahead**：标签严格 forward；PIT join 各源 t_available；t_decision<t_realized。
- **selected-on-test**：Train/Select/Report 三分离；阈值/early-stop/特征选择只在 Select；report 一次性。
- **overlap leakage**：net_swing 5日窗 + 步长3日 → purge + embargo=5日。
- **scaler leakage**：logistic/LightGBM scaler fit train-fold only。
- **model overfitting**：logistic 先做线性天花板；LightGBM early-stop 监控 Select；SHAP 仅诊断。
- **metric misreporting**：PR-AUC+期望收益+coverage+skill score，不用 accuracy；命名带角色(hpval_/report_)。
- **baseline honesty**：先验+free-rule+v1+指纹四基线同 report 索引；skill score>0 才上线。
- **regime/存活**：报 by-regime；声明数据集 regime；标注存活偏差。
- **小样本诚实**：block bootstrap CI + McNemar 配对 + trial count 惩罚。
- **live parity**：同特征/lag/缺失策略；陈旧数据/kill-switch 规则。
- **建模后验证**：阶段9 V1–V11 多门签核（泄漏复审→三分离→重叠→基线→模型族→度量→漂移→复现→经济回测→live parity→签字门），`verify_model.py` 自动化 + 人工判断，任一门失败不得上线。
