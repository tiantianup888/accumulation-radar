## 0. 验证目标（基于新架构重新定义）

回答五个问题：
1. 双容器能起、能健康运行、SQLite 共享卷可读写？—— **部署可用性**
2. 信号正确落库、价格快照不丢、SQLite WAL 并发不冲突？—— **数据持久化正确性**
3. API 返回与库内数据一致、PnL 实时计算公式算对？—— **API 与 PnL 正确性**
4. 落库的信号数据（费率/OI/市值/横盘）与第三方源一致？—— **信号数据正确性**（沿用原方案 L1/L3）
5. 积累 N 天后，各策略信号的 PnL 分布、胜率、命中率是否合理？—— **信号有效性（实盘积累版）**

**关键转变**：原方案因无历史数据无法验证"信号有没有用"；新架构通过持久化实盘信号 + 跟踪 PnL，**用积累的实盘数据逐步给出有效性证据**。跟踪期 5 天，积累 1/3/5 天三档结论。

## 4. L3 — 信号数据正确性（与第三方源交叉核对）

> 这一层沿用原方案思路，但数据来源从"本次推送文本"改为"signals 表"，更结构化。

### 4.1 核对流程
1. 跑 1 小时，让 `signals` 表积累一批信号
2. 随机抽 20 条信号（覆盖三个策略）
3. 对每条信号的 `funding_rate / oi_delta_6h / mcap / sideways_days` 用第三方源实时核对
   - 费率 → 币安官网费率页 / Coinglass
   - OI 变化 → Coinglass OI 24h 变化
   - 市值 → CoinGecko `coins/markets`
   - 横盘天数 → TradingView 日线图肉眼
4. 算误差，超阈值标红

### 4.2 误差阈值
| 指标 | 可接受误差 |
|------|-----------|
| 费率 | 绝对 < 0.01% |
| 市值 | 相对 < 15% |
| OI 变化% | 绝对 < 3% |
| 横盘天数 | < 10 天 |

### 4.3 已知风险点（沿用原方案，实时确认）
- **R1 — OI 单位**：`openInterestHist.sumOpenInterestValue` 是 USDT 还是币数？用 BTC 实时值与 Coinglass 对照。若错，`MIN_OI_USD=2M` 门槛失效，影响所有 OI 过滤。
- **R2 — 市值接口稳定性**：`bapi/.../marketing/symbol/list` 跨小时稳定性 + 与 CoinGecko 误差。
- **R4 — `CMCCirculatingSupply` 字段**：dump 原始 JSON 确认存在性，影响市值 fallback。

### 4.4 产物
- `docs/verify/cross_check.csv`：`signal_id, 指标, 库内值, 第三方值, 误差, 是否通过`
- `docs/verify/data_audit.md`：R1/R2/R4 结论

## 5. L4 — 信号有效性（实盘积累版，替代历史回测）

> 这是新架构带来的能力：**用系统自己积累的信号 + PnL 给出有效性证据**。样本量随时间增长。

### 5.1 阶段性目标（与跟踪期同步5天）
| 积累时长 | 样本量（估） | 能给的结论 |
|---------|-------------|----------|
| 1 天 | ~10-30 信号 | 仅印象分，无统计意义 |
| 3 天 | ~30-100 信号 | 初步分布、趋势性观察 |
| 5 天 | ~50-150 信号 | 初步统计结论（胜率/命中率可信） |

### 5.2 关键指标（由 API `/strategies/{name}/stats` 实时算出）
| 指标 | 含义 | 评判 |
|------|------|------|
| `avg_unrealized_pnl_pct` | 平均未实现盈亏 | >0 且 > 基准(全市场等权) |
| `win_rate_pct` | 胜率 | >50% 算可用 |
| `hit_rate_50pct` | 曾涨≥50% 占比 | >20% 算有信号价值 |
| `hit_rate_100pct` | 曾涨≥100% 占比 | >5% 算抓得到大肉 |
| `avg_max_low_pct` | 平均最大回撤 | 评估风险 |
| `分策略对比` | chase vs combined vs ambush | 埋伏应领先追多（提前期更长） |

### 5.3 基准对照
- **随机抽样基准**（最优决策）：每小时从全市场随机抽 50 个币采价，与信号组同等跟踪期内算 PnL 对比
- 不做全市场等权（开销大），50 币/小时样本足够给基准印象
- 基准由 API 额外端点 `/benchmark` 出

### 5.4 已知偏差（必须披露）
- **幸存者偏差**：跟踪期内下架的币按最后一次有效价计 PnL 并标 `delisted`，统计时单独披露 delisted 比例，不剔出（剔出会美化结果）
- **样本自选择**：信号本身是按某条件筛的，与随机基准对比时需注意"信号币本身流动性/市值分布"与基准不同
- **市场环境偏差**：5 天的单边市场不能代表全周期

### 5.5 产物
- 每天自动生成 `docs/verify/effectiveness_day_{N}.md`：当日样本量、PnL 分布、与基准对比
- 5 天后汇总 `docs/verify/effectiveness_final.md`
