## 二、匹配方法


对每个币滑动14日窗口（每3日一个快照，同币命中后冷却5日），与指纹库每条指纹计算相似度：

```
相似度 = 0.4×价格形状相关 + 0.3×量比形状相关 + 0.3×行为签名匹配率
```

**关键发现**：纯形状相似是噪音（样本外期望-0.49%，胜率42.8%），因为下跌币和起飞前形状太像。**形状相似 + 正确行为签名**才产生正期望。经切片验证，分化器是「波动率收敛 + 止跌企稳 + 低位」三签名。


## 三、锁定生产配置


```
similarity >= 0.65  AND  波动率收敛(converge)  AND  止跌企稳(standstill)  AND  低位(pos<0.3)
```

## 七、结论与生产决策


1. **指纹库方法有效**：形状相似+行为签名(收敛+止跌+低位)的样本外90天期望 **+3.16%**，胜率53.6%，肥尾捕获5.2%，跌>10%仅7.8%，RR0.58。这是本项目首个通过样本外验证的正期望信号。

2. **关键分化器是签名而非形状**：纯形状相似样本外-0.49%(噪音)；叠加「收敛+止跌+低位」后转正且随相似度阈值单调增强(0.55→+1.87%, 0.68→+3.32%)，证明非偶然。

3. **止跌企稳是最关键签名**：区分「洗盘后反转」(止跌→涨)与「趋势下跌/下跌中继」(探底→继续跌)。探底长下影反而是负期望(-2.39%)。

4. **生产决策**：上线 `fingerprint` 策略，配置 similarity>=0.65 + 收敛+止跌+低位。保留 `sideways_acc`(v1)作对照。

5. **局限**：库仅19条指纹(来自90天83个拉盘事件中合格者)，样本有限；C超跌反弹已排除但仍有少量假阳性(跌>10%占7.8%)；需实盘持续验证与库扩充。
